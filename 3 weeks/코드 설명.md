## 2. 파일 경로 정의 및 데이터 초기화

```python
def manage_mars_inventory():
  input_file = 'Mars_Base_Inventory_List.csv' # 원본 데이터 파일
  output_csv = 'Mars_Base_Inventory_danger.csv' # 위험 물질 저장용 CSV
  output_bin = 'Mars_Base_Inventory_List.bin' # 전체 데이터 저장용 이진 파일

  inventory_data = [] # 데이터를 담을 리스트
  header = '' # CSV의 첫 줄(제목)을 담을 변수
```
* 프로그램 내에서 사용할 파일 경로를 변수로 지정하여 코드의 **재사용성**과 **가독성**을 높였습니다.


## 3. 데이터 로드 및 전처리 (File I/O & Parsing)

```python
    try:
      with open(input_file, 'r', encoding='utf-8') as f:
        header = f.readline().strip() # 첫 번째 줄(헤더)을 읽고 공백을 제거한다.

        for line in f:
            line = line.strip()
            if not line: continue # 빈 줄 건너뛰기

            fields = line.split(',') # 쉼표를 기준으로 데이터 분리

            try:
                fields[4] = float(fields[4]) # 5번째 열(인화성)을 실수형으로 변환한다
            except (ValueError, IndexError):
                fields[4] = 0.0 # 변환 실패 시 0.0으로 처리

            inventory_data.append(fields)
```

* `with open`: 파일 열기 및 닫기를 자동으로 관리하여 메모리 누수를 방지합니다.
*  `try-except` **(중첩)**: 데이터 중 숫자가 아닌 값이 섞여 있거나 데이터가 누락된 경우에도 프로그램이 중단되지 않도록 **안정성**을 확보했습니다.

## 4. 데이터 분석: 정렬 및 필터링

```python
    # 인화성 지수(index 4)를 기준으로 내림차순 정렬
    sorted_inventory = sorted(inventory_data, key=lambda x: x[4], reverse=True)

    # 인화성 지수가 0.7 이상인 위험 물질만 필터링
    danger_list = [item for item in sorted_inventory if item[4] >= 0.7]
```

* `sorted` **&** `lambda`: 익명 함수인 람다를 활용하여 정렬 기준을 유연하게 설정했습니다.

* **리스트 컴프리헨션**: 파이썬다운(Pythonic) 방식을 사용하여 간결하고 효율적으로 데이터를 필터링 했습니다.

## 5. 결과 저장 (Exporting)

### 5.1 CSV 형식 저장 (텍스트 기반)

```python
    with open(output_csv, 'w', encoding='utf-8') as f:
        f.write(header + '\n')
        for item in danger_list:
            line = ','.join([str(val) for val in item]) # 데이터를 다시 쉼표로 연결한다
            f.write(line + '\n')
```

* 필터링된 위험 물질 데이터를 범용적인 CSV 형식으로 저장하여 다른 프로그램(엑셀 등)에서도 확인할 수 있도록 했습니다.

### 5.2 Pickle 직렬화 저장 (이진 기반)

```python
    with open(output_bin, 'wb') as f: # 'wb': Write Binary 모드
        pickle.dump(sorted_inventory, f)
```

* **객체 직렬화**: 데이터 구조 자체를 바이너리로 저장하여, 추후 데이터를 불러올 때 별도의 변환 과정 없이 파이썬 리스트 객체로 즉시 복구할 수 있습니다.
