## 1. 모듈 임포트 및 설계 의도 (Imports)
```Python
import random
from datetime import datetime
import os
```

random: 화성 기지의 가변적인 환경 데이터를 시뮬레이션하기 위해 사용합니다. => 화성의 불확실한 환경 데이터를 시뮬레이션하기 위해 난수를 생성합니다.

datetime: 로그 기록 시 '언제' 측정되었는지 시각 정보를 제공합니다. => 센서 데이터가 측정된 정확한 시점을 기록하여 시계열 데이터를 형성합니다.

os: 로그 파일의 존재 여부와 크기를 체크하여, 파일 시스템을 안전하게 제어하기 위해 사용합니다. => 로그 파일(`sensor_log.txt`)의 존재 여부와 크기를 확인하여 시스템의 안정성을 확보합니다.

구조적 이점: 하드코딩(Hard-coding)을 피하고 클래스 상단에 범위를 모아둠으로써, 환경 정책이 변경될 때 코드 본문을 건드리지 않고 이 딕셔너리만 수정하면 되는 유지보수성을 확보했습니다.

## 2.2 __init__ (초기화 및 캡슐화)

```Python
def __init__(self):
  self._env_values = {
    'mars_base_internal_temperature': 0,
    'mars_base_external_temperature': 0,
    'mars_base_internal_humidity': 0,
    'mars_base_external_illuminance': 0,
    'mars_base_internal_co2': 0,
    'mars_base_internal_oxygen': 0
  }
  self._log_file = 'sensor_log.txt'
```

변수 보호: self._env_values와 같이 변수명 앞에 언더바(_)를 붙여, 외부에서 직접 데이터를 수정하는 것을 방지하는 캡슐화(Encapsulation) 개념을 적용했습니다.

데이터 일관성: 모든 센서 키(Key)를 미리 정의하여 데이터 구조의 일관성을 유지합니다.


# 3. 핵심 로직 상세 해설 (Methods)
## 3.1 set_env(): 데이터 생성 로직
이 메서드는 단순히 값을 만드는 것이 아니라, 실제 센서의 특성을 반영하도록 설계되었습니다.

```Python
def set_env(self):
  ranges = list(self._RANGES.values())
  for i, key in enumerate(self._env_values.keys()):
    low, high = ranges[i]
    val = random.uniform(low, high)
    precision = 3 if 'co2' in key else 2
    self._env_values[key] = round(val, precision)
```

범위 매칭: _RANGES에 정의된 순서대로 루프를 돌며 데이터를 생성합니다.

데이터 정밀도 제어:

일반 데이터(온도, 습도 등)는 소수점 2자리까지 표시합니다.

이산화탄소(co2) 데이터는 미세한 변화가 중요하므로 **조건문(if 'co2' in key)** 을 통해 소수점 3자리까지 생성하도록 정밀도를 차별화했습니다.

## 3.2 get_env(): 데이터 반환 및 로그 자동화
이 프로그램에서 가장 복잡하고 중요한 로직이 포함된 부분입니다.

```Python
def get_env(self):
  is_empty = not os.path.exists(self._log_file) or os.path.getsize(self._log_file) == 0

  with open(self._log_file, 'a', encoding = 'utf-8') as f:
    if is_empty:
      header = 'timestamp,' + ','.join(self._env_values.keys()) + '\n'
      f.write(header)

    now = datetime.now().strftime('%Y-%m-%d %H:%M:%S')
    values_str = ','.join(map(str, self._env_values.values()))
    f.write(f'{now},{values_str}\n')

  return self._env_values
```

지능형 헤더 생성:

```Python
is_empty = not os.path.exists(self._log_file) or os.path.getsize(self._log_file) == 0
```

이 조건문을 통해 파일이 아예 없거나, 파일은 있지만 내용이 비어있을 때만 컬럼명(Header)을 작성합니다. 이는 로그 파일의 데이터 무결성을 지켜줍니다.

파일 입출력 최적화: with open(...) as f 구문을 사용하여 파일 기록 후 자동으로 닫히도록 설계하여 메모리 누수를 방지했습니다.

CSV 포맷팅: ','.join()을 사용하여 엑셀이나 데이터 분석 도구(Pandas 등)에서 즉시 활용 가능한 CSV(Comma-Separated Values) 형식으로 저장합니다.

# 4. 프로그램 실행 흐름 (Execution Flow)
프로그램이 시작되면 다음과 같은 순서로 제어권이 이동합니다.

```Python
if __name__ == '__main__':
  ds.DummySensor()
  ds.set_env()
  current_data = ds.get_env()

  print('현재 화성 기지 환경 데이터:')
  for key, value in current_data.items():
    print(f'{key}: {value}')
```

인스턴스 생성: ds = DummySensor()를 통해 객체를 메모리에 할당합니다.

환경 설정: ds.set_env()를 호출하여 random 라이브러리가 6개의 센서값을 독립적으로 생성합니다.

데이터 확정 및 저장: ds.get_env()를 호출하면:

저장소(sensor_log.txt)를 확인합니다.

필요 시 헤더를 작성합니다.

현재 시각과 생성된 데이터를 한 줄로 기록합니다.

최종 데이터를 딕셔너리 형태로 반환합니다.

출력: 메인 루프에서 반환된 데이터를 반복문을 통해 사용자 화면에 보여줍니다.

# 5. 설계 시 고려한 PEP 8 가이드라인
Indent: 4개의 공백(Spaces)으로 들여쓰기를 통일했습니다.

Spacing: 연산자(=, +) 주변과 쉼표(,) 뒤에 적절한 공백을 두어 가독성을 높였습니다.

Naming: 클래스는 PascalCase, 메서드와 변수는 snake_case를 사용하는 파이썬 표준 관례를 철저히 준수했습니다.
